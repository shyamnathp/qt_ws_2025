Supporting new languages in Qt?

Option 1: Rewrite Qt in New Languages

Option 1: Rewrite Qt in New Languages

  • ‚ùå Massive effort
  • ‚ùå Loss of maturity
  • ‚ùå Fragmentation risk

Key Insight: Rewriting Qt is not practical.

- **Massive effort**: Rewriting a framework as mature as Qt would take years of development and testing. It would require significant resources and could delay innovation. - **Loss of maturity**: Qt has decades of optimizations and stability. A rewrite would lose this maturity, and the new version might lack feature parity. - **Fragmentation risk**: Creating language-specific versions could split the community and lead to inconsistent behavior across implementations. - Conclude by emphasizing that this approach is not feasible for a framework as established as Qt.

Option 2: Create Language Bindings for Qt

Option 2: Create Language Bindings for Qt

  • Preserve Qt's maturity and stability
  • Incremental adoption path
  • Proven approach (e.g., PySide6, QtJambi)

Question: What powers PySide6? Meet the unsung hero...

- Preserve Qt's maturity: Language bindings allow us to leverage Qt's decades of stability and optimizations. - Incremental adoption: Developers can adopt bindings gradually without disrupting existing workflows. - Proven approach: PySide6 is a successful example of this strategy. - Conclude by emphasizing that this approach is practical and scalable.

QtPythonScripting


#include <QtPythonScripting/QPythonClassType>
#include <QtPythonScripting/QPythonScriptEngine>

int main(int argc, char *argv[])
{
    QPythonScriptEngine engine;
    auto pointType = engine.registerType<QPoint>();
    pointType.template registerConstructor<int, int>();
    pointType.registerMethod("x"_ba, &QPoint::x);
    pointType.registerMethod("y"_ba, &QPoint::y);
    pointType.registerMethod("manhattanLength"_ba, &QPoint::manhattanLength);
    pointType.registerMethod("setX"_ba, &QPoint::setX);
    pointType.registerMethod("setY"_ba, &QPoint::setY);

    constexpr auto script = R"(
point = QPoint(1, 1)
point.setY(2)
print("Point: ", point.x(), ", ", point.y(), ", manhattanLength=", point.manhattanLength())
)"_L1;
    engine.execute(script);
    return 0;
}

Does that mean I do not need PySide6?


    from PySide6.QtGui import QGuiApplication
    from PySide6.QtQml import QQmlApplicationEngine
    from QtBridges import AutoQmlBridge

    class StringModel:
        def add_string(self, new_string: str) -> bool:
            print(f"Added: {new_string}")
            return True

    # Register with QML
    app = QGuiApplication(sys.argv)
    engine = QQmlApplicationEngine()

    string_model = StringModel()
    AutoQmlBridge(string_model)

    qml_file = "main.qml"
    engine.load(str(qml_file))

    if not engine.rootObjects():
        sys.exit(-1)

    exit_code = app.exec()
    del engine
    sys.exit(exit_code)
