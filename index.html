Traditional Approaches

  • Modify source code. (e.g., SWIG, JNI, etc.)
  • Manual binding maintenance
  • Framework-agnostic solutions

Shiboken's Way

  • Non-intrusive binding generation
  • Automatically adapts to new Qt features and classes
  • Qt-specific optimizations
- JNI requires modifying C++ code with special JNI macros and types - Python/C API requires extensive boilerplate in C code - SWIG needs special directives in source files - Emphasize the importance of not modifying Qt's source code - Highlight automation benefits for large frameworks - Qt-specific optimizations - Efficient conversion from Qt containers to Python sequences, Signal Slot mechanism etc.

Does that mean I do not need PySide6?


    from PySide6.QtGui import QGuiApplication
    from PySide6.QtQml import QQmlApplicationEngine
    from QtBridges import AutoQmlBridge

    class StrModel:
        def add_string(self, new_str: str) -> bool:
            print(f"Added: {new_str}")
            return True

    # Register with QML
    app = QGuiApplication(sys.argv)
    engine = QQmlApplicationEngine()

    string_model = StrModel()
    AutoQmlBridge(string_model)

    qml_file = "main.qml"
    engine.load(str(qml_file))

    if not engine.rootObjects():
        sys.exit(-1)

    exit_code = app.exec()
    del engine
    sys.exit(exit_code)
